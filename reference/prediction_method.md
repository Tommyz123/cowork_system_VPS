## 六步涌现流水线（方法骨架）

```
① 抽实体        → 只抽"现实中真实能公开发声/互动的主体"，禁止抽象概念
② 建知识图谱    → 实体+事实+关系连成图；图 = 信息差和可见性的载体
③ 长角色        → 每个实体只从图谱取它自己的真实事实，长成 persona
③.5 造环境      → 有规则的互动场：可见性规则 + 动作集 + 回合时钟（独立一环，不可糊进碰撞）
④ 多轮碰撞      → 每轮激活一批角色各自选动作发言；发言回写图谱
⑤ 提取预测      → 上帝视角报告：必调检索、必引角色原话、禁用自身知识、不出概率数字
```

## 七条铁律（每条 = 一个真实踩过的坑）

**铁律一：先夯实种子事实，数据不实 = 儿戏**
每条事实标确信度（✅核实 / 🟡待核 / ⚠️存疑 / ❓未知）；关键变量未知就在预测里明确标"未知"，禁止脑补。种子不准则涌现全错。（坑2：和解破裂日期误判、冻结范围搞反）

**铁律二：信息差必须物理造出来**
每个角色只拿它身份**该接触**的信息子集，并明确写出它的盲区（角色信息分发表）。同一批事实喂所有角色+贴立场标签 = 假碰撞，结论早被预设。（坑1，最致命）

**铁律三：角色从真实事实长出，不是立场标签**
persona 由该实体在图谱里自己的事实生成；只抽真实主体，排除"诉讼走向/市场情绪"这类假实体。（坑4）

**铁律四：环境是独立一环**
没有"有规则的互动场"（谁看到谁 + 能做什么动作 + 回合时钟），碰撞无从发生，角色只会各说各话。（坑5：漏掉 OASIS）

**铁律五：不报概率数字**
禁止拍脑袋甩"40%""50-55%"。输出 = "哪些力量推向哪个方向、依据是谁的什么原话"。（坑3）

**铁律六：先理解原理再执行**
跑预测前必须能讲清：信息源怎么找、信息差怎么造、角色怎么各不相同。讲不清 = 没懂，先补课。（坑6，主公最强调；已入全局 CLAUDE.md"先理解再执行"）

**铁律七：对外结果必带定位免责**
每次对外给预测结果，必带一句：**"这是机制推演沙盘，不是基于今日数据的实时预测；角色编的具体数字是 roleplay 非真数据。"** 避免把模拟数字当真。

## 能力边界（诚实定位，2026-06-08 读完原版源码后评估）

- 订阅版复刻了 **~35-40% 工程能力、~80% 原理价值**：六步骨架/信息差物理造/引原话不报概率全对；差距在图谱不回写、无 RecSys 可见性、6-8 角色 vs 原版几千 agent。
- **定位：小规模快速推演工具（圆桌推演），不是大规模涌现引擎。**
- 升级路线（按序）：① 接 GraphRAG 让碰撞回写图谱 → ② 扩 agent 规模 + RecSys 式可见性 → ③ 上帝视角动态注入变量。
