## 三、实验设计规格 v0.1 六块（CC 初稿）

**第1块·唯一问题（锚）**
评分能否在"市场没反应"时把"后来跑赢大盘的票"排前面 = Q1排序力 + Q2择时力。
明确**不回答**"能否赚钱"、不做对冲基金级。评分价值 = 排序力，不是盈亏。

**第2块·样本单位**
一笔 = 〔股票 + 信号日 + 持有期 + 评分版本〕四件套绑死。冷却期 = 一只票成样本后 N 天（=持有期）内再入选不算新样本。

**第3块·主指标**
固定 20 交易日相对收益 = 个股收益% − 同期IWM收益%；>+3%对 / <-3%错 / 中间平。持有期事前定死（20日官方，5/60/90日辅助观察）。另记相对行业ETF版本。

**第4块·对照组**
A 同池低分票（后25%纸面跟踪不下单）答Q1排序力 / B 随机抽25只比"评分vs瞎选" / C 纯IWM基准。

**第5块·全样本入库 + 分层**
所有样本入库只标状态（filled/unfilled/no_cash/control_low/control_random）绝不删。三层物理隔离：客观数据层（更新）/ 冻结快照层（信号价/6维分/LLM prompt输出/版本，永不改）/ 主观判断层（verdict/归因，单独分区+标判断时间+来源+依据，绝不回写覆盖前两层，演进=新增带时间戳记录非改旧）。统计分离：客观收益统计不依赖归因填没填。

**第6块·影子打分**
v1正式版决定下单，v2影子版只记账不下单纸面跟踪，攒N月后同批票同段市场PK谁排序更准，胜者转正。

- 第1轮审"改进方案" → `trading/notes/codex_risk_review_20260629.md`（约40条；核心判断"方案偏工程改良，没把实验设计写死"）

### Codex 二轮逮住的 3 个致命硬伤（v0.1 确实漏，须补）

1. **🔴 "市场没反应"无可操作定义**：整份规格灵魂是"叙事先于价格埋伏"，却没定义"没反应"。若入选当天已大涨，后续跑赢只是动量延续非埋伏，Q2择时力被污染。→ 须加"信号日前价格反应标签"（信号日前5天相对大盘涨跌/有无跳空），分"已反应/未反应"。
2. **🔴 主统计用信号层 vs 成交层没定义**：主实验应用"信号层收益"——所有高分票按冻结信号价算，不管成没成交；交易执行（成交率/滑点）单独报告。**顺手化解 OPG 17% 成交率纠结：成交率属交易执行，不该影响评分排序力主统计。**
3. **🔴 信号价取哪个价没定义**：20日收益对入场点敏感，须钉死（建议"信号后下一可交易日开盘价"），防人为挑有利价。

### 重要不致命（v0.2 补，做最简版）

4. ±3%三分类丢排序信息 → 主指标改连续相对收益+高低分组均值差/秩相关，±3%只当可读标签
5. 低分组风险分布可能不同（天然小盘/差流动性）→ 加行业/市值粗分桶平衡检查（**不上因子模型，Codex自己也强调**）
6. 随机组抽样边界 → 钉死"从45候选抽=测评分"+固定随机种子
7. 影子转正规则 → 连续两评估窗赢v1且非单月/单票极端值驱动才转正

### Codex 对 CC 4个担心点的回应（更好解法，已采纳）

- 持有期20日/阈值±3%：不致命，但须预注册成唯一主口径，禁辅助窗口推翻主结论
- 样本量门槛：第一版定"最低可解释门槛" = 至少6个月 + 每组至少150有效信号样本 + 覆盖多行业/市场状态，才下v1结论
- 归因防事后诸葛亮：拆"信号前归因"（下单前冻结，解释评分）vs"结果后复盘"（标post-mortem，只复盘不回写）
- 标注负担：靠字段分层——主实验只依赖客观收益+冻结评分（机器自动算），归因是辅助非成立条件，post-mortem可抽样

## 六、可迁移到第2层的底层原则

虽然本规格是第1层的，但以下**通用实验严谨性原则**可搬到第2层（趋势主线）：
- 相对收益判对错（扣大盘 beta，区分 alpha vs 运气）
- 可追溯（每个判断能回溯依据）
- 数据/判断分层（客观数据零主观、判断单独分区标时点）
- 不事后诸葛亮（事前写验证点，事后只填兑现没兑现）
- 样本量门槛（攒够才下结论）

**不可迁移**（第1层全自动专属）：6维评分通胀修复、自动下单、影子打分、季度扫描节奏。
