# 第2层提速攒样本方案（前向纸仓·影子判断记账）v0.2

> **【这是什么】** 趋势主线（第2层）的提速验证方案——用 intraday 纸账户放开出手门槛，把「六维打分→登记→跟踪→对答案」循环从一年 1-2 圈提到每月 3-5 圈，快速验证"这套趋势方法方向对不对"。
> **【为什么做】** 第2层有验证死锁：验证哲学要求多样本才能下结论，但出手纪律（等摊牌+双闸）把样本掐到一年 1-2 个。主公 2026-07-01 拍板：纸账户阶段放开出手，快速收集数据，但数据必须零污染（"不然一点用都没有"）。
> **【为什么这样设计】** 核心=解耦「出手门槛」与「记录纪律」：放开的只是出手频率，锁死的是每笔的记录流程。真钱双闸纪律原样不动。v0.2 按 Fable5 对抗审核（6致命+7重要+5次要，存档 [fable_audit_第2层提速方案_20260701.md](fable_audit_第2层提速方案_20260701.md)）全部修法重构，核心变化=对答案拆三本账、结论门槛按独立风险簇、verdict 机械落库、扫描长名单留痕。
> **【给谁看】** 未来 AI + 主公。配套：`第2层认输线.md`（裁判）/ `趋势判断手册.md` v1.1（尺子）/ `趋势地图_2026Q2.md`（第0批来源）。

## 一、主公三条死约束

1. **提速攒样本**：快速有数据，验证方法方向对不对
2. **数据必须有用**：不能是垃圾数据/噪音
3. **零污染**：可追溯 + 数据/判断分层 + 不许事后改口径

## 二、被验证的三个假设（三本账，物理分开，禁止混算）

| 账 | 验证什么 | 谁进账 | 判定方式 |
|---|---|---|---|
| **A·排序力（主假设）** | 六维分数能否把"后来跑赢的趋势"排在前面 | **全部样本**（passed/borderline/rejected，含第0批） | passed 组 vs rejected 组的 6 个月相对 SPY **连续收益**分布比较（组间均值差+秩相关）。**不逐笔判对错**——rejected 的唯一角色是对照分布（六维低分=证据不足，不是预测跑输，AWS 压线案例为证） |
| **B·方向判断准确率** | 登记的方向判断（看多/看空/观望）准不准 | 只有**登记了冻结方向判断**的样本（第0批全部排除） | 三向口径写死：看多=6个月相对SPY ≥+5% 对；看空=≤−5% 对；观望=\|x\|<5% 对。±5% 只是本账的可读标签，不是主口径 |
| **C·上下车有效性** | 下车信号/保命线触发的纸面卖出，是躲过下跌还是卖飞 | 只有实际触发过下车动作的纸仓 | 下车价 vs 之后 3 个月该标的走势 + 同期 SPY，独立统计 |

**统一规则**：A/B 两账一律按**信号层买入持有**计算（3m/6m/12m 窗口，不受中途下车影响；下车只进 C 账）——持有区间事前钉死，堵"按实际持有还是按窗口"的事后选择。

## 三、核心规则

### 规则 1｜账户：intraday 纸账户（已核实），物理隔离
- **2026-07-01 实测**：intraday 账户 ACTIVE，权益 $1,006,475 全现金，0 持仓 0 挂单——干净，无 ghost 残留，无需 reset
- **统一金额：$25,000/笔**（权益÷40 取整；passed/borderline 建仓用，40 个槽位够 2 年节奏）
- 新脚本锁死 `account='intraday'`（镜像第1层 `assert_p9_account`，方向相反）；第1层代码/表/cron 零改动
- **范围声明**：统一金额=本实验**不验证**手册 B 节的仓位档位规则，结果不得外推到仓位管理

### 规则 2｜出手：打分即登记，cohort 三分
- 每条完成六维打分的趋势候选，**无论得分**都登记进 DB（判断冻结）：
  - `passed`（得分率≥80% 且无一票否决）→ 建纸仓 $25k
  - `borderline`（60-80%）→ 建纸仓 $25k
  - `rejected`（<60% 或一票否决命中）→ **只做信号层记账，不建纸仓**（对照分布不需要执行层；消除组间执行差异+省资金+少分支）
- **方向=看空的样本不做纸面做空**：只信号层记账，标 `no_position`

### 规则 3｜标的：每条趋势一个代理标的，选择规则+板块基准都冻结
- 优先行业 ETF，无合适 ETF 选龙头股；登记选择理由+选择日期；只做美股
- **板块基准规则（写死，登记时冻结基准代码）**：代理=个股 → 基准=其 GICS 一级 sector ETF；代理=细分行业 ETF → 基准=一级 sector ETF；代理=一级 sector ETF → 本项标 N/A
- 辅助口径"相对板块基准"只观察（区分选对趋势 vs 只是选对板块），不推翻 A/B 账主结论

### 规则 4｜事前冻结登记（防污染闸①）
每笔登记时写死、由 SQLite trigger 禁止 UPDATE/DELETE：
- 六维打分快照（每分带出处+出处日期；缺数据留空不计分母）+ **手册版本号**（尺子会迭代，跨版本样本不混算）
- **risk_cluster 风险簇**（必填：该趋势的命脉挂在哪个共同因子上，如 AI-capex链/国防/GLP1……）——同簇同窗口的样本在 B 账合并计 1 个有效判断；**结论门槛=≥8 个独立风险簇**，"30 笔"只是数据量描述不是结论门槛
- **trigger_source 触发来源**（必填：摊牌信号/W7嗅探/heatmap升温/配额扫描/主公喂/backfill）——统计时分层报告，配额扫描进来的候选单独看，防对照组被"本不该打分的东西"稀释
- 方向判断（看多/看空/观望，可空=不进 B 账）+ 一句话依据 + 验证点 + 下车信号（rejected 组不设下车信号）
- 入场价口径=登记日**下一交易日开盘价**（信号层）；实际纸仓成交价另记在执行层，主统计不用

### 规则 5｜上下车照手册真执行（只对纸仓）
- 下车信号/保命线（成本-25%/峰值-30%）触发 → 走手册信号检查表 → 纸面卖出+登记原因 → 进 C 账
- 纸仓默认持有至 12 个月终判后了结（或下车信号提前触发）

### 规则 6｜真钱边界 + 纸验证结果的预注册映射（2026-07-01 主公拍板）
- 本方案全部动作限 intraday 纸账户；真钱出手仍守原纪律（FERC 终局+capex 双闸+主公拍板）
- **预注册映射（写死，堵"结果坏就说仅供参考"）**：6 个月主判若 **A 账 passed 组相对 rejected 组无优势** → 自动触发：**暂停第2层真钱新开仓 + 启动根因复盘**；复盘找到有依据的改进（如某一维系统性钝化）→ 修完可解除暂停重新攒样本再验。失败的直接后果=真钱闸门收紧，自动生效不容商量；但非一次失败判死刑（对齐"先改进再认输"）。与 `第2层认输线.md` 互为引用

## 四、防污染闸（七道，v0.2 补供给端和裁决端）

| # | 污染形态 | 防线 |
|---|---|---|
| 1 | 事后诸葛亮（结果出来改判断） | 冻结表 SQLite trigger 禁 UPDATE/DELETE；复盘只 INSERT 新记录 |
| 2 | 数据/判断混写 | 三层分表：judgments冻结 / signal_prices+paper_trades数据 / verdicts判断 |
| 3 | 只记赢家（入库后） | 全样本入库，含 rejected，一条不删 |
| 4 | **选择性供给（入库前，v0.2 新增）** | **每月扫描长名单留痕**：扫到但未进打分的候选+一句话淘汰理由，全部入 trend_scan_longlist 表——幸存者偏差的上游堵口，也是 false negative 复盘的原料 |
| 5 | 口径漂移 | 三本账口径+持有区间+结论门槛+真钱映射全部本文档预注册 |
| 6 | **裁判自审（v0.2 新增）** | verdict 由脚本按预注册口径**机械判定直接落库**，BB 无权改判；BB 只可另行 INSERT discrepancy 标记（附证据）报主公裁决 |
| 7 | 编分 | 每分带出处+日期；缺数据留空；主公不定期抽查打分表 |

## 五、数据存储（trading.db 新增 5 表，与第1层表零交集）

```
trend_judgments     判断冻结层（trigger 禁 UPDATE/DELETE）：登记日/趋势名/代理标的+类型+理由+选择日/
                    板块基准/六维分JSON/总分/得分率/手册版本/cohort/risk_cluster/trigger_source/
                    方向/依据/验证点/下车信号/备注
trend_signal_prices 数据层（每 judgment 一行，INSERT once）：信号入场价(次日开盘)/SPY基线价/板块基线价/来源
trend_paper_trades  数据层（仅 passed/borderline）：judgment_id/订单id/建仓日/成交价/股数/
                    平仓日/平仓价/平仓原因/status——纯客观，出场字段允许事实回填
trend_verdicts      判断层（trigger 禁 UPDATE/DELETE，只 INSERT 可滚动多条）：judgment_id/账(A|B|C)/
                    窗口(3m|6m|12m)/原始收益/相对SPY/相对板块/verdict/判定日/判定依据(含基线价+数据源)/
                    是否机械(默认1)/discrepancy标记
trend_scan_longlist 供给留痕层：扫描日/候选名/一句话淘汰理由/是否进入打分
```

## 六、自动化分工（AI出草稿·人把关 + 机械裁决）

| 环节 | 谁做 | 说明 |
|---|---|---|
| 候选供给 | BB 每月扫一批 + W7 + heatmap + 主公喂；**长名单全留痕** | 3-5 条是目标带宽非硬指标，禁止凑数 |
| 六维打分+登记 | BB 人工跑（不 cron 化，打分质量是命根），主公抽查 | 每批登记后 Discord 报打分表 |
| 纸仓下单 | `trend_paper_trade.py`（CLI，锁死 intraday） | 每笔下单前 Discord 报"买什么/为什么" |
| 对答案 | `trend_verdict_check.py`（cron 周一 17:15，幂等）：到窗口的样本自动拉价、按口径**机械判定落库**，Discord 报结果 | BB 无权改判，只可标 discrepancy |
| 周检联动 | 周一周检顺手过纸仓下车信号（预计+10分钟；超载时降级顺序：先砍档案叙事更新、后砍下车检查——下车检查不可砍） | 档案只给重点纸仓对象开块，全样本轨迹不进档案（防撑爆） |
