# 趋势追踪档案（Trend Tracking Dossier）

> **用途**：趋势主线的「病历本」——每个进池对象记下①为什么进池②进池时的基线，然后顺着这条理由的节奏每周追踪轨迹，让判断可回溯、可复盘。
> **价值边界（防高估）**：本档案的价值是「帮主公把判断练准」，**不是「替主公赚钱」**。它治「轨迹丢失」，不治「数据质量」；源头数据仍是 agent 搜来的，垃圾进=垃圾出。档案做了 ≠ 策略有效——策略有效与否只有真实战绩说了算。
> **铁律**：①只记事实+逻辑状态，**绝不写「该买/该卖」**；②进池/出池决策权永远在主公；③数据带来源，搜不到就留空（沿用《趋势判断手册 v1.1》纪律）；④数据/判断分层隔离（出生档案=事实写死，轨迹=判断单独标时点）。
> **创建**：2026-06-19（阶段1：结构化档案 + 半自动更新；阶段2 跑顺后评估 Agent 化）
> **追踪节奏**：每周一随趋势周检（cron）顺手更新轨迹，只追加不删改。

---

## 📐 档案结构说明（怎么读）

每个对象分两块：
- **📋 出生档案**：进池时写死，**永不修改**——记「为什么进池 + 当时的基线数据」
- **📈 追踪轨迹**：每周追加一行，**只增不改**——记「关键数变化 + 逻辑还成不成立 + 离触发远近」

**⚠️ 加新对象必带字段（自动写入依赖）**：每个对象出生档案第一行必须写
`- **追踪代码**：XXX（自动写入用）`（个股=股票代码；趋势=代理股代码，如电力=GEV）。
`dossier_autowrite.py` 每周从档案**自动解析**所有对象+追踪代码来追加轨迹，**对象清单不写死在代码里**——
所以加新对象只改本档案（照格式写一段+带追踪代码），脚本零改动自动纳入；缺这字段的对象会被跳过并告警。

轨迹表五列（**数据与判断物理分层**——2026-06-22 改，目的：数据列零主观，AI 读档案不会把旧判断当事实滚雪球）：
- **📊数据**：纯客观事实（股价/估值/距高/涨跌/与上周对比），**一个主观词都不准进**
- **🧭逻辑状态**：进池理由还成立吗？`✅成立 / 📈强化 / ⚠️松动 / ❌瓦解`（这是判断）
- **🧭判断**：纯推断（成立度/位置好坏/离首仓闸门 FERC/capex 或下车信号多远），**不放生数字**
- **数据源**：yfinance / 趋势地图 / 手动，便于追溯
- ⚠️ **铁律**：📊数据列与🧭判断列**永不混写**——数据会变、判断跟着变，分列才能干净追溯对比
