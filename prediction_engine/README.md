# 通用预测引擎（订阅版 MiroFish 复刻）

> 建立：2026-06-08 | 来源：复刻 github.com/666ghj/MiroFish 的六步涌现流水线
> 性质：用 Claude 订阅额度（`claude -p` headless）替代付费 LLM/Zep API，零 API 费跑多智能体社会模拟预测。

## 这是什么

输入「一份事实台账 + 一个要预测的问题」，自动跑完六步涌现流水线，输出走向预测报告。
**引擎层零硬编码**——实体/图谱/角色/发言/预测全部由 claude 从输入生成，换任何题目代码一行不改。

## 用法

```bash
python3 predict.py <事实台账文件> "<要预测的问题>" [回合数] [输出目录]
```

例（大麻案）：
```bash
python3 predict.py cases/organic_blooms/facts.md "诉讼及December队列到2026年底走向" 3
```

换题目（财报/选举/政策…）：只换 facts 文件 + 问题，引擎不变。

## 六步涌现流水线（复刻 MiroFish）

| 步 | 做什么 | 实现 |
|----|--------|------|
| ① 抽实体 | 从facts抽真实能发声的主体（禁抽象概念） | claude sonnet |
| ② 建知识图谱 | 抽关系三元组连成图（信息差的载体） | claude sonnet |
| ③ 长角色 | 每个实体从图谱取回相关三元组，长成人设+知识子集 | claude sonnet |
| ③.5 造环境 | 可见性(各角色知识子集)+动作集(post/reply)+回合时钟 | 引擎逻辑 |
| ④ 多轮碰撞 | 每轮按序激活角色，看feed后发言，实时回写 | claude haiku |
| ⑤ 提取预测 | 上帝视角读完整feed，引各角色原话推演，不报百分比 | claude sonnet |

## 关键设计

- **大脑后端**：`claude -p`，cwd=/tmp（避免加载 CLAUDE.md 浪费 token）。
- **角色扮演破法**：claude 默认拒绝"假装真人发帖"，改用「生成标注 FICTIONAL_SIMULATION_DATA 的合成训练数据」格式即放行。
- **信息差是物理造的**：每个角色的 `knows` 来自它在图谱上能取回的三元组子集，有明确 `blind`（盲区），不是同一批事实贴标签。
- **预测铁律**：引原话、标缺口、不报概率、不脑补未知。

## 已知边界

- 订阅版天花板：小规模（4-8角色、2-5轮），跑不出真版几十agent大规模涌现。
- 真版有 Zep GraphRAG 自动持续回写图谱；本引擎的图谱是一次性抽取，碰撞发言未回写图谱（简化）。
- 角色发言用 haiku 求速度，质量可换 sonnet 提升。
