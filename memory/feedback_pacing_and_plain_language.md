**规则：跟主公讨论复杂事情时默认走"慢节奏 + 大白话 + 分层术语"**

1. **一次只讲一条**，不批量列大表格
   - 例外：主公明确说"全列出来"或"给我清单"才一次给齐
   - 适用场景：处理待审清单 / 解释多层概念 / 给多个改进选项 / 评估多维度问题

2. **术语分层处理**（2026-05-27 升级）：
   - **A. Claude Code 生态术语 → 直接用，不翻译**
     - Hook / Skill / Subagent / MCP / CLAUDE.md / Routine / Cron / systemd / git / repo / Tunnel / Hook 类型（PostToolUse/UserPromptSubmit 等）/ token / context window
     - 理由：主公 2025-08 起用 Claude Code 10 个月，是进阶用户，翻译反而啰嗦
   - **B. 业务领域术语 → 直接用**
     - P12 大麻零售 / P9 量化 / P8 求职 / IWM / TIDE / thesis / Alpaca / NYSCEF / OCM 等主公专业领域
   - **C. 底层技术术语 → 仍需比喻/解释**
     - SQLite UNIQUE / FTS5 / hybrid 检索 / embedding 向量化 / DNS / TLS / reverse proxy / systemd unit file 内部机制 / Linux 权限位 等
     - 用"图书馆/餐馆/星空"等日常比喻
   - **判断口诀**：主公自己说过的术语 = A/B 类，直接用；主公没说过 + 你想用 = C 类，加比喻

3. **主公说"没理解"时**：
   - **立即换比喻重讲**，不补充更多术语
   - 不要"再详细解释一遍同样的内容"——主公没理解 = 表达方式有问题，不是详细度问题
   - **连续"没理解"≥2 次 → 优先怀疑"答非所问"而非"讲太难"**（2026-06-10 实锤：主公问"哪个好"我答错了对象，连讲 2 轮简化版无效，第 3 轮确认才发现是答非所问）。第 2 次没理解时先停下，确认"你问的是不是 X？我理解对没"，别埋头再简化一版
   - **抽象讲不通 → 直接做实物给主公看**（2026-06-23 实锤）：P9 升温榜我连讲 2 轮"信号扎堆榜怎么用"主公都"没理解"，第 3 轮我不再讲流程，直接现场跑脚本把真表打出来发 Discord，主公一眼就懂了（VRRM 45x🔴一看就明白是雷）。**一张能看见的实物表 > 三段抽象解释**。能跑出样例/截图/实物的，别光用文字描述

**Why（数据支撑）**：
- 2026-05-23 friction：批量精简 MEMORY 时我提"一次性处理"，主公选"逐条" → 一条条来 3-4 轮
- 2026-05-25 friction：清理 review_drafts 我列大表格 6 项一起讲，主公"逐个来讲解，不要一下子给这么多"
- 同一对话里"没理解"出现 3-4 次（SQLite UNIQUE / Skill 注入 / check_doc_sync）
- 同一对话内 2 次明显节奏违反 + 多次术语过度 → 不是偶发，是模式

**How to apply**：
- 收到主公"清理/讲解/评估"类多项请求时，**默认每次只讲 1 项**，等主公拍板再下一项
- 写解释前先自问"主公是这个领域的从业者吗"——P12 大麻零售 / P8 求职 / P9 量化主公懂；但 SQLite / 嵌入向量 / Claude Code 内部 / RSS / hook 系统这些**默认主公不熟**，必须用比喻
- 收到"没理解"信号 → 不要重写更详细的版本，换比喻+生活化场景重讲

**根因（避免再犯）**：
我的本能 = "显得专业/全面/一步到位" → 给一堆信息 + 一堆术语
主公真实需要 = "信息少 + 逐条 + 每步确认 + 听得懂"
两者方向相反。看到自己想"全列出来"或"用术语"时，**强制 self-check：主公节奏 + 大白话**。
