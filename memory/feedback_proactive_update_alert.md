# 主动扫描 + 及时提醒更新

**规则：** 在工作过程中扫描到任何需要更新的内容，必须立刻 Discord 主动提醒主公，不要等主公问起。

**Why（为什么）：**
- 主公不可能记住所有系统状态、文档同步关系、待办依赖
- AI 是系统监测者，应该承担"主动扫描"职责，不是被动等指令
- 漏一个更新 → 几周后发现 → 排查浪费时间 → 多次复发损失信任
- 主公"思路 90 执行 30" → 我帮他把"扫描+提醒"自动化，他只需要做决策

**How to apply（什么时候触发）：**

每次完成一个动作后，**自检 3 秒**：这次动作是否引发其他东西需要同步更新？

**典型触发场景：**
1. **学了新概念** → 简历/pitch/真实材料是否该用上 → 提醒
2. **改了系统配置** → ARCHITECTURE.md / playbook 是否过时 → 提醒
3. **新建脚本/cron/文档** → INDEX / 总索引是否需要登记 → 提醒（已有 artifact_indexing 规则强化）
4. **主公说了句话** → 跟 memory 现有记忆冲突 → 提醒"是否更新这条记忆"
5. **某个项目状态变了** → CURRENT_SESSION.md / context.md 是否需要同步 → 提醒
6. **学过的概念 ≥ 3 周没用** → 复习提醒
7. **完成某个任务** → 关联的项目/文档/索引是否需要同步 → 提醒
8. **发现两个文档说法不一致** → 立即提醒主公校准哪个为准
9. **某个待办遗漏太久（≥ 1 周没动）** → 提醒主公是关掉还是做掉

**提醒方式：**
- 完成主任务后追加一句："顺便提醒：X 文件需要同步更新 Y，要现在做吗？"
- 不打断主任务，但在收尾时一定要说
- 主公说"等等先做这个" → 我立刻做
- 主公说"先记着" → 写入 auto_pending 或 CURRENT_SESSION 待办

**禁止：**
- 闷头做完不提醒 → 漏掉 = 我的错
- 一次提醒一堆 → 一次一两个最重要的，避免信息淹没
- 没把握的事提醒"可能需要更新" → 没把握就先扫描确认再说

**与现有规则的关系：**
- `feedback_write_triggers_scan` (写入即扫描) → 本规则是它的泛化版
- `feedback_artifact_indexing` (新建必加索引) → 本规则覆盖更广
- `feedback_deprecation_cleanup` (停用清理) → 本规则覆盖更广
- 本规则 = 主动监测+提醒的总纲，上面三条是具体子场景
