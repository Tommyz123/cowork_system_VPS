**Cowork 层只记录以下大事，不记录细节操作：**
- 新建或修改桌面级文件（context.md、cowork_log.md 等）
- 启动 Claude Code 任务（记录目录和任务描述）
- 浏览器重要操作（打开网页、提交表单等）
- 安装软件包或执行系统级脚本
- 任何报错或失败

**Claude Code 内部的操作**（读取代码文件、修改数据库、运行爬虫等）由 Claude Code 自己的 activity.log 处理，Cowork 不重复记录。

**日志格式（一行）：**
`[YYYY-MM-DD HH:MM] 类型 | 对象 | 结果/说明`

**类型图标：** ✏️新建 | 📝修改 | 🗑️删除 | 📁新建文件夹 | 🚀任务 | 📦安装 | 🌐浏览器 | ❌报错 | 📋总结 | 💾保存进度

**Why:** Cowork 层靠 AI 自律记大事，细节操作不需要单独记录。activity.log 曾尝试用 Hook 实现但已删除（信息量不足，价值有限，2026-03-31 确认）。

**How to apply:** 一个主公意图完成后，先记日志再回复主公。一个意图 = 一条，写最终结果。失败也要记录，写明错误信息。

1. 更新 `CURRENT_SESSION.md` 对应项目块（状态/完成内容/下一步）
2. 追加 `💾保存进度` 到 `cowork_log.md`
两步缺一不可。
