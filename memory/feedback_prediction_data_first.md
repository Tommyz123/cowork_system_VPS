主公对"先跑预测、后发现数据错"的做法明确定性为"儿戏""不准确"，要求：**数据没准确，禁止开始预测。**

**Why:** 2026-06-07/08 帮主公预测纽约大麻2023零售队列(Organic Blooms v OCM, Index 904497-24)走向时，我连跑多版预测，但事实地基一直是错的——先把"局部冻结(只冻没选址申请)"误当"整个12月队列冻结"，又把原告(其实是要求停止抢跑的合规方)误当"被冻结的受害者"，还误判"和解2026-01已死"(实为Blackmark单线破裂)。等于流沙上盖楼，跑再多都是白跑。主公追问后选B："你数据都没有准确就开始预测了。"

**How to apply:** 任何预测/推演/研判类任务，正确顺序是反的——
1. 先建「事实台账」：每条事实标 ①内容 ②来源(文件名/NYSCEF文档号/搜索出处) ③确信度(已核实/存疑/缺口)
2. 明确列出缺口和存疑项，先补齐(主公有内部材料则喂、公开可搜则搜)
3. 数据干净、缺口补到位后，再决定是否预测、怎么预测
数据不准时任何预测都是负价值(给错误的安全感)。

**附带的方法论自省(同次对话):** LLM分多个子agent演角色报概率，独立性是假的(本质同一模型同样盲区)，概率数字是拍脑袋无基率支撑=伪精确，且无独家信息差。真预测要有历史基率/明确推理链/真实信息差。
