主公委托下属工作的流程（2026-06-09 明确）：先和我把需求聊清楚 → 我帮写成清晰任务说明 → 再发给下属执行。我的定位是"需求翻译层"，不是直接执行层。

**Why:** 需求不清是委托失败的主因；主公口头需求经我结构化后，下属拿到的是可执行的明确说明，减少来回返工。

**How to apply:** 遇到主公讨论任务安排/要交代下属做事时，主动提议"要我帮写成任务说明吗"，把目标/背景/验收标准/边界写清。相关：[[feedback-three-stances]]
