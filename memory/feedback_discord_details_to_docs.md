多轮技术讨论（对抗审核、方案迭代、系统设计）向主公发 Discord 时：**技术细节全部收进文档，消息只给「结论 + 主公需要拍板的那一个问题」**。

**Why**：2026-07-01 P9 第2层方案审核讨论实锤——BB 连发几轮高密度清单（6致命/三本账/风险簇等术语堆叠），主公反馈"有没有越说越乱了"。根因：把"列出所有发现"当成汇报完整，实际上主公在手机上需要的是决策入口，不是工作量证明。已有 [[feedback_pacing_and_plain_language]]（大白话+术语分层）没覆盖"多轮讨论中细节该往哪放"这个场景。

**How to apply**：
1. 发消息前先问自己："主公真正需要决定的是哪一件事？"——只发那件事，细节留在文档里给要看的人
2. 审核报告/方案变更这类产出：文档存档路径 + 一句话结论 + 待拍板项，三样即可
3. 收到"乱了/没懂"类反馈时，立即用三句大白话收拢：①要干啥 ②为什么 ③结果怎么用，末尾只留一个决定问题
4. 与 [[feedback_discord_long_doc]]（长文档分段贴）区分：那条管"长内容怎么发"，本条管"多轮讨论中该不该发细节"——默认不发，收进文档。
