主公计划购买 Mac mini 作为24小时在线个人AI助理服务器，仍在考虑 M4 vs M5。**倾向等 M5（2026-06-06）**——理由是不急；但已想清楚 M5 比 M4 强的点（性能/能效/AI算力）大部分对主公"遥控+存储+AI走Claude云端"的用法用不上，真瓶颈是内存非芯片代次。

**选配结论（2026-06-06 深聊更新，覆盖旧24G底线判断）：**
- **推荐 M4(或等M5) / 32GB / 1TB**。32G 是平台多开/并发的舒适线，1TB 给数据增长留 4-5 年空间。两者都是 Mac 焊死不可换/SSD 可外挂。
- **不建议为本地模型多花钱上 48G**——主公有 Claude Max（额度单人够用），AI 主力走云端；本地模型只是锦上添花。
- **本地AI需求清单 + 适配判断（今天聊出）：** ✅Whisper语音转写(3-5G,轻松) / ✅小语言模型≤14B(7-8B吃5-8G) / ✅SD前端配图(SD1.5/SDXL,32G够,慢点无所谓) → 这三类32G全能跑；❌视频模型/重度生图(需英伟达CUDA独显，Mac没独显跑不动且慢)→走云端GPU租用或独显PC，跟内存大小无关。
- 旧判断（2026-04-22）：曾以24GB为底线（可跑Qwen2.5 32B Q4）。现因主公明确"小模型为主+有Max"，结论更新为32G足够，不必为跑大模型堆内存。

**长期定位（2026-06-06）：** cowork 要从轻量工具长成"私人助理平台"，数据/功能只增不减；核心诉求=记录+提醒。Mac mini 是这个平台的本地承载。新方向：想做"能听会说的语音助理"（Whisper耳朵+TTS嘴巴，全本地、隐私不出门），等机器到手后整套搭。
