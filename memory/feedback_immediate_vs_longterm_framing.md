**规则：** 给主公推任何"3 选 1 / 优先级建议 / 该做哪个"类决策时，**第一动作**是问"这是想立即推动，还是长期方向规划？"——两种场景主公的判断框架完全不同：

| 场景 | 主公的判断顺序 |
|---|---|
| **立即推动** | 1️⃣ 难度 / 可行性 → 2️⃣ ROI 速度 → 3️⃣ 战略契合度 |
| **长期方向** | 1️⃣ 战略契合度（哪怕重）→ 2️⃣ 独家壁垒 → 3️⃣ 现在不重要 |

**关键反直觉：** "重 / 难做" 在长期方向场景**不再是缺点**——因为本来就是等条件成熟才动，重的活到时候就是那阶段的正经活。

**Why:**
2026-05-27 02:00-02:10 EDT 讨论"跨实例通讯能赚啥钱"，我给主公 3 个路径（A 大麻店 SaaS / B 法律订阅 / C 求职作品）。第一次猜主公会选 B，理由是"难度中等 + 立刻能 differentiate"——结果**猜错**，因为我默认按"立即推动"框架推。主公加约束"我不是马上做"后，我重猜 A（最贴 P12 主线），主公确认对。

主公原话总结："立即做 → 看难度+ROI；长期方向 → 看战略契合。"

**How to apply:**
- 主公问"X / Y / Z 选哪个"或"该做啥"时，**先反问**："你是想接下来一两周就动手，还是先想清楚长期方向？"
- 别凭直觉默认"立即推动"框架——主公**更常见的模式**是"先讨论长期方向，等条件触发再动"
- 长期方向场景下，**不要再用"太重 / 太难"打折候选方案**，要按"3 年后哪个最贴战略终点"排序
- 配合 `feedback_methodology` 使用：长期方向选好之后，仍然走"小步起步+触发升级"路径（不一蹴而就）
