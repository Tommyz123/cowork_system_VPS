P9 量化系统的去留判断标准是「先改进，再认输」，由主公于 2026-06-02 确立，已写入 `playbooks/p9_trading.md`（策略定位区块后）。

三条：
1. **比就比狠的**：每次评估和 IWM(小盘) + SPY(大盘) + QQQ(科技) 三个指数一起比，且看风险调整后(Sharpe)。绝不只挑最弱的小盘股(IWM)比、赢了就自我感觉良好。
2. **跑输 ≠ 关掉，而是 = 找原因改进**：复盘根因(选股逻辑/参数/板块)再想办法跑赢；禁止亏了乱调参数骗自己。
3. **认输门槛很高**：必须多次迭代、跨多季度、风险调整后依然稳定跑输，才承认没机会、买指数。

**Why**：主公明确纠正过我两次——①不能只和小盘股(IWM)比，长期跑输任何主要指数就该反思「为什么不直接买指数」；②但「跑输」不等于「立刻关掉」，而是「想办法跑赢，真没机会我才认」。我早先的「一刀切/判死刑」框架被否定。

**How to apply**：评估 P9 表现时，永远三指数同比+Sharpe，不要 cherry-pick。讨论是否该停 P9 时，默认立场是「找改进空间」而非「该关了」；只有在已多季度多次迭代后仍稳定跑输时，才支持认输。注意基准坑：DB 列名 `spy_entry` 实际存的是 IWM 价格(config.py BENCHMARK_SYMBOL="IWM")。
