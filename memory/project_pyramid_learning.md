# 《金字塔原理》学习项目

**书：** 中文版（汪洱、高愉译，Barbara Minto 原著）
**学法：** AI 讲解 + 测试 + 实战，真懂才进下一章
**启动：** 2026-05-26（另一对话启动，2026-05-26 19:17 EDT 交接到 cowork 架构）

## 教学规则（5条 + 元规则）
1. 流程：分小节讲 → 每节确认 → 章末出 3 道练习（概念+识别+实战）→ 通过才进下一章
2. 风格：严格、诚实、不附和；同样错犯第二次必须指出
3. 写作练习：AI 只给框架/素材/诊断，主公自己动手写，不替写
4. 落地：每章学完后用真实场景（简历/求职材料）练习再进下一章
5. 称呼：叫"主公"，中文为主，技术词中英混用

**元规则（2026-05-31 主公确认）**：这套学习流程要"盯死执行"——每次学习自动按 5 条走，无需重新提醒。主公会在学习过程中持续给反馈（节奏/难度/举例方式等），我的动作：① 当场调整执行 ② 把反馈更新进上面的教学规则 ③ 之后一直按更新版执行。一句话：**流程盯死、反馈即改、改完记住**。

**学习目标（2026-06-01 主公明确）**：主公的目标是把这套技能"**学透**"——不是看懂，是能不假思索地用出来（形成肌肉记忆）。他准确判断本书是"工具书/方法论"，每条都是拿来用的工具。**教学重心因此偏向落地**：每章学完必须用主公**真实材料**（求职信/pitch/给老板的提案/Discord 长消息）反复练，同一工具在不同材料上练好几遍，练到他自己写东西时手自动按方法走。不能停在"测验过了"就进下一章。

**"会用"4 级标准（2026-06-01 共识，验收用）**：L1 看懂（认得出）→ L2 提示下能用 → **L3 无提示自动用对（=学透目标）** → L4 能挑错/能教。**L3 硬指标**：连续 3 份不同真实材料，主公不看笔记、我不提醒，写出来结构都基本对（结论先行+归纳带证据+SCQA），只需我小修。到此才算该章"透"，才进下一章。

**我的训练承诺（2026-06-01 主公委托"训练我到学透"）**：① 盯进度，不让主公停在"看懂"就推进 ② 主动出落地练习，不等主公想起 ③ 狠批不放水，但每次给明确"还差哪一步" ④ 记住并反复盯易错点直到改掉。角色分工：主公动手写，我盯/批/纠/追踪到 L3。

**深练 vs 看懂的分层策略（2026-06-01 我给的判断，主公询问后）**：不是每章都死磕 3 份（太慢会拖垮）。**核心工具** → 深练到 L3，因为天天用、是真求职竞争力；**边角内容** → 看懂即可不强求练。**由我负责判断哪些深哪些浅（主公明确要求：我必须心里有数才能帮到他，进边角章节要主动提示"看懂就行"）**，主公不用自己分类。理由：本书是工具书+主公有求职 pitch 真实刚需，看懂就推进会"每章都懂一点、都用不出来"。

**全书深浅地图（2026-06-01 我给主公的导航，后续教学按此）**：
🔴 核心-深练到 L3（骨架，写材料必用）：① 结论先行 ② MECE ③ 纵向疑问—回答 ④ 归纳 vs 演绎 ⑤ SCQA 序言 ⑥ 概括分组论点（第4章，怎么提炼好的上层句子）。前 5 个第 1-2 章已学。
🟡 中等-看懂+练 1 次：序言常见模式（第3章）、演绎细节、解决问题之界定问题/结构化分析（第7-8章，对商业分析有用但非写作高频）。
⚪ 边角-看懂就过：咨询行业案例分析、序言细分变体、演示/PPT 排版（第9-10章）。

**⚠️ 主公易错点（讲后续章节注意）**：主公反复把"纵向关系"理解成"下层解释上层"，但正确是"下层回答上层勾起的疑问"。第一小节测验纠正过一次，整章测验又犯第二次。"解释"= 单向补充说明（被动），"回答疑问"= 读者主动发问、我来答（对话）。后续凡涉及纵向关系，确认主公用"回答疑问"而非"解释"。
**另一易错点**：归纳列理由时主公倾向笼统"我有 A 和 B"，需提醒升级成"做过 A（证据）、做过 B（证据）"的并列带证据条目。
